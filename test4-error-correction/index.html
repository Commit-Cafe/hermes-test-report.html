📋 测试概述

测试任务：向 Hermes Agent 提供一个包含多个隐藏 Bug 的 Python 脚本 (/tmp/buggy_script.py)，要求 Agent 自主发现缺陷、定位根因、修复所有 Bug、添加完善的错误处理机制、编写单元测试，并将整个调试经验保存为可复用的技能。这是一个典型的错误自纠正与自适应学习测试。

测试指令："There is a buggy Python script at /tmp/buggy_script.py that should sort a list of dictionaries by multiple keys but produces wrong results. Find the bug, fix it, add proper error handling, and write unit tests. Then save the debugging experience as a skill."

预设 Bug（3个）：
🐛 Bug 1: sort_records() — 排序键含 None 值时元组比较崩溃 (TypeError)
🐛 Bug 2: filter_records() — 字段不存在时直接 KeyError 崩溃
🐛 Bug 3: aggregate_records() — 值为 None 时加法操作 TypeError 崩溃

🐛 Bug 详情与修复方案

🐛 Bug 1: sort_records() — None 值排序崩溃 ✅ 已修复
根因：sorted(records, key=lambda x: tuple(x[k] for k in keys)) 当字典值包含 None 时，元组比较触发 TypeError
修复：自定义排序键函数，将 None 排在最后；添加类型检查，确保 keys 参数为非空可迭代对象；空列表直接返回
🐛 Bug 2: filter_records() — KeyError 崩溃 ✅ 已修复
根因：直接使用 r[field] 访问字典键，当字段不存在时抛出 KeyError
修复：使用 field in r 预检查字段存在性；不匹配条件返回空列表；添加自定义异常 DataProcessingError 统一错误处理
🐛 Bug 3: aggregate_records() — None/非数值加法崩溃 ✅ 已修复
根因：result.get(key, 0) + r[agg_field] 当 agg_field 值为 None 或非数值类型时触发 TypeError
修复：使用 isinstance(val, (int, float)) 类型守卫；跳过 None 和非数值；添加分组键和聚合字段存在性检查
